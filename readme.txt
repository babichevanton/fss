Прежде всего необходим установленный MySQL server. Необходимо создать и проинициализировать базу данных приложения. Для 
этого из папки DBscripts в терминале нужно выполнить:
mysql < drop.sql
mysql < create.sql
mysql < initialize.sql

Для запуска приложения из директории Hibernate (в ней должен лежать файл build.xml) нужно выполнить в терминале команду
ant run -Daction=<action> [-Darg1=<arg1>] ...
Возможные варианты action:
1)storeflight
  arg1 = number (строка, произвольная)
  arg2 = length (целое неотрицательное число)
  arg3 = airline (строка, имя существуюущей в базе данных авиалинии. Доступно 2: AirBerlin и Lufthansa)
  arg4 = dptr_airport (строка, имя существуюущего в базе данных аэропорта. Доступно 4: Domodedovo, Vnukovo, Tegel, Schonefeld)
  arg5 = arr_airport (строка, имя существуюущего в базе данных аэропорта. Доступно 4: Domodedovo, Vnukovo, Tegel, Schonefeld)
  Сохраняет новую запись в таблицу Flight
  
2)authorization
  arg1 = login (строка, произвольная)
  arg2 = password (строка, произвольная)
  Ищет клиента с данной парой логин/пароль и выводит его имя, если находит. Иначе пишет "Wrong login/password"
  
3)flights
  без параметров
  Выводит все авиарейсы с информацией о ценах и свободных местах
  
4)one_flight
  arg1 = idPartFlight(целое неотрицательное число, id существующего в базе данных рейса. Доступно 1:5, но лучше 1 или 2)
  Выводит информацию о состоянии мест на данный рейс

5)order_seat
  arg1 = idPartFlight(целое неотрицательное число, id существующего в базе данных рейса. Доступно 1:5, но лучше 1 или 2)
  arg2 = row (целое неотрицательное число, номер ряда на борту. Какие доступны, см. соотв. запись в таблице. Помогает one_flight)
  arg3 = symbol (строка, литера места в ряду. Помогает one_flight)
  arg4 = orderway (строка, способ бронирования. Доступно 2: order и reserve)
  Приобретение места. Недоделано, но статус места в таблице меняет.

6)seats
  Для тестирования отображения таблиц в POJO-классы. Бесполезен.
